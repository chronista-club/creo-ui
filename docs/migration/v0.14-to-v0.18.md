# Migration: creoui v0.14 → v0.18

> **Audience**: v0.14 系列を install している consumer (creo-memories / vantage-point / fleetstage 等)。 v0.18 が **5 tier convention 完全統一 (token + component attribute 両層) の最終 release** で、 以降の release から convention drift しない安定 surface に到達。

## 概要 — 4 release window

v0.14 → v0.18 は **4 つの BREAKING release** を含むが、 consumer 視点では一度に jump する path が推奨です:

| from | to | 主要変更 |
|---|---|---|
| **v0.14.0** | v0.15.0 | 5 新 token (scrim / scrim-modal / container / grid / icon) + 14 component に `prefers-reduced-motion` guard 完全適用 + dialog/drawer backdrop の scrim token 化 |
| v0.15.0 | v0.16.0 | spacing / container / grid / icon の token name を **`xs/s/m/l/xl` の 5 tier convention に部分統一** (BREAKING、 partial) |
| v0.16.0 | v0.17.0 | 残 token (margin / radius / shadow / typography.size / typography.display) を **5 tier convention に完全統一** (BREAKING、 token layer final) |
| v0.17.0 | **v0.18.0** | Component **attribute axis** も 5 tier 統一: `[data-size="sm/md/lg"]` → `[data-size="s/m/l"]`、 `[data-padding="sm"]` → `[data-padding="s"]`、 `[data-elevation="sm"]` → `[data-elevation="s"]` (BREAKING、 attribute layer final) |

実 consumer は v0.15 / v0.16 / v0.17 を skip して **v0.14 → v0.18 直行** を推奨。 中間 release は publish 直後に partial state で問題発覚、 production 採用は無く、 以下の guide も「最終 v0.18 reality」 を target にして書かれています。

## Soft-landing — token alias (`token-shim.css`) で CSS variable は段階移行可能

v0.16-v0.17 の token rename は **token-shim.css の legacy alias 経由で旧 var 名 (`--margin-md` 等) が動き続ける**。 consumer は:

```ts
import 'creoui/tokens.css'
import 'creoui/token-shim.css'  // ← これを足せば v0.14 系の var(--margin-md) 等が今のままで動く
```

新 5 tier 名 (`var(--margin-m)`) への置換は段階的に進められ、 完了後に `token-shim.css` を抜く Phase C が migration goal。

ただし **v0.18 component attribute は HTML markup 層で alias できない** ので、 `data-size="md"` → `"m"` の HTML 書換が必要 (CSS shim では救えない)。

## Token rename — 完全 table (v0.14 → v0.18)

CSS variable / JS const / Swift identifier / Rust const の **4 representation** で同 rename が適用されます。

### Dimension scale (5 tier 完全統一)

```diff
# Spacing
- var(--spacing-sm)  →  var(--spacing-s)       /*  8px  */
- var(--spacing-md)  →  var(--spacing-m)       /* 18px  */
- var(--spacing-lg)  →  var(--spacing-l)       /* 24px  */
  var(--spacing-xs)  →  var(--spacing-xs)      /* unchanged */
  var(--spacing-xl)  →  var(--spacing-xl)      /* unchanged */

# Margin (v0.17 で rename)
- var(--margin-sm)   →  var(--margin-s)        /* 16px  */
- var(--margin-md)   →  var(--margin-m)        /* 18px  */
- var(--margin-lg)   →  var(--margin-l)        /* 40px  */

# Radius (v0.17 で rename、 none / full 据置)
- var(--radius-sm)   →  var(--radius-s)        /*  8px  */
- var(--radius-md)   →  var(--radius-m)        /* 15px  */
- var(--radius-lg)   →  var(--radius-l)        /* 22px  */
  var(--radius-none) →  var(--radius-none)     /* unchanged */
  var(--radius-full) →  var(--radius-full)     /* unchanged */

# Shadow (v0.17 で rename、 none 据置)
- var(--shadow-sm)   →  var(--shadow-s)
- var(--shadow-md)   →  var(--shadow-m)
- var(--shadow-lg)   →  var(--shadow-l)

# Typography size (v0.17 で rename)
- var(--typography-size-sm)  →  var(--typography-size-s)   /* 14px */
- var(--typography-size-md)  →  var(--typography-size-m)   /* 16px */
- var(--typography-size-lg)  →  var(--typography-size-l)   /* 18px */

# Typography display (v0.17 で rename)
- var(--typography-display-sm)  →  var(--typography-display-s)   /* 32px */
- var(--typography-display-md)  →  var(--typography-display-m)   /* 44px */
- var(--typography-display-lg)  →  var(--typography-display-l)   /* 56px */
```

値 (= 物理 dimension) は **据置**、 名前のみ変更。

### 命名 convention 原則 (v0.17 から strict)

すべての 5 段階 dimension scale token は:

```
xs  /  s  /  m  /  l  /  xl
```

の 5 tier に統一。 中央 `m` が default、 上下 2 段ずつ拡張。 `sm` / `md` / `lg` の Tailwind 流命名は **完全廃止**。 `radius.none` / `radius.full` / `shadow.none` のような **special semantic** は 5 tier の外に残置。

この原則は `docs/foundations/principles` の原則 01 にも articulate されています。

### Layout primitive token (v0.16 で 5 tier 化、 v0.17 でも touch なし)

v0.15 で追加された Layout primitives は v0.16 で既に 5 tier rename 済。 v0.17 で再 rename はありません。

```css
/* Layout container (v0.16 で 5 tier rename + xs 追加) */
var(--layout-container-xs)    /* 480px、 v0.15 で追加した sm/md/lg/xl の代替 */
var(--layout-container-s)     /* 640px */
var(--layout-container-m)     /* 768px */
var(--layout-container-l)     /* 1024px */
var(--layout-container-xl)    /* 1280px */

/* Layout grid auto-fit (v0.16 で 5 tier rename + xs/l 追加) */
var(--layout-grid-col-min-xs) /* 120px */
var(--layout-grid-col-min-s)  /* 160px */
var(--layout-grid-col-min-m)  /* 220px */
var(--layout-grid-col-min-l)  /* 280px */
var(--layout-grid-col-min-xl) /* 320px */

/* Typography icon (v0.16 で 5 tier rename + xs/s 追加) */
var(--typography-icon-xs)     /* 16px */
var(--typography-icon-s)      /* 24px */
var(--typography-icon-m)      /* 40px */
var(--typography-icon-l)      /* 64px */
var(--typography-icon-xl)     /* 96px */
```

### Surface scrim token (v0.15 で追加、 dialog / drawer backdrop)

```css
var(--color-surface-scrim)        /* 40% black、 drawer / side sheet */
var(--color-surface-scrim-modal)  /* 50% black、 dialog (中央 modal) */
```

dialog.css / drawer.css の `::backdrop` で `rgba(0, 0, 0, ...)` 直書きを置換、 token SSOT 原則に準拠。

## Component attribute migration (v0.18)

v0.18 で 17 component の `data-size` / `data-padding` / `data-elevation` attribute を 5 tier に統一。 consumer は **HTML markup を書換** する必要があります (CSS variable alias では吸収不可)。

### 全 attribute rename (v0.18)

```diff
# Button / Input / Dialog / Drawer / Popover / Tabs / Table / Pagination / Badge /
# Skeleton / Stepper / Progress / Spinner / Breadcrumbs / Timeline / Avatar
- <button class="creo-btn" data-size="sm">              → <button class="creo-btn" data-size="s">
- <button class="creo-btn" data-size="md">              → <button class="creo-btn" data-size="m">
- <button class="creo-btn" data-size="lg">              → <button class="creo-btn" data-size="l">
- <input  class="creo-input" data-size="md" />          → <input class="creo-input" data-size="m" />
- <dialog class="creo-dialog" data-size="lg">           → <dialog class="creo-dialog" data-size="l">

# Card data-padding
- <article class="creo-card" data-padding="md">         → <article class="creo-card" data-padding="m">
- <article class="creo-card" data-padding="lg">         → <article class="creo-card" data-padding="l">

# Header data-elevation
- <header class="creo-header" data-elevation="sm">      → <header class="creo-header" data-elevation="s">
- <header class="creo-header" data-elevation="md">      → <header class="creo-header" data-elevation="m">
```

### 影響 component (17)

| component | attribute | values (v0.17 → v0.18) |
|---|---|---|
| Button / Input / Dialog / Popover / Tabs / Table / Pagination / Skeleton / Progress / Spinner | `data-size` | `sm/md/lg` → `s/m/l` |
| Drawer | `data-size` | `sm/md/lg/xl` → `s/m/l/xl` |
| Avatar | `data-size` | `sm/md/lg/xl` → `s/m/l/xl` |
| Badge / Breadcrumbs / Stepper / Timeline | `data-size` | `sm/md` → `s/m` |
| Card | `data-padding` | `sm/md/lg` → `s/m/l` |
| Header | `data-elevation` | `sm/md` → `s/m` (`none` 据置) |

### Bulk sd pattern (consumer codebase 側)

```bash
# data-size
sd 'data-size="sm"'      'data-size="s"'      src/**/*.{html,tsx,jsx,vue,svelte}
sd 'data-size="md"'      'data-size="m"'      src/**/*.{html,tsx,jsx,vue,svelte}
sd 'data-size="lg"'      'data-size="l"'      src/**/*.{html,tsx,jsx,vue,svelte}

# data-padding (Card のみ)
sd 'data-padding="sm"'   'data-padding="s"'   src/**/*.{html,tsx,jsx,vue,svelte}
sd 'data-padding="md"'   'data-padding="m"'   src/**/*.{html,tsx,jsx,vue,svelte}
sd 'data-padding="lg"'   'data-padding="l"'   src/**/*.{html,tsx,jsx,vue,svelte}

# data-elevation (Header のみ)
sd 'data-elevation="sm"' 'data-elevation="s"' src/**/*.{html,tsx,jsx,vue,svelte}
sd 'data-elevation="md"' 'data-elevation="m"' src/**/*.{html,tsx,jsx,vue,svelte}
```

> **note**: `data-variant` (primary / ghost / 等)、 `data-placement` (top / bottom / left / right)、 `data-shape` (text / circle / rect)、 `data-cols` (1/2/3/4/6/12 + auto-* 5 tier 既統一)、 `data-state` (default / error / disabled) 等は **5 tier 命名に該当せず touch なし**。

## JS const migration

```diff
- import { SpacingMd, MarginLg, RadiusSm, ShadowMd, TypographySizeMd } from 'creoui/tokens.js'
+ import { SpacingM,  MarginL,  RadiusS,  ShadowM,  TypographySizeM  } from 'creoui/tokens.js'
```

camelCase identifier も同 rename rule (`Md` → `M`、 `Lg` → `L`、 `Sm` → `S`)。

## A11y reduced-motion (v0.15 で完全対応)

v0.15 で 14 全 component が `@media (prefers-reduced-motion: reduce)` を guard。 consumer 側で **追加作業なしで自動的に WCAG 2.1 SC 2.3.3 + SC 2.3.1 準拠**。 spatial transform (switch thumb slide / button press / table sort indicator 等) もすべて停止します。

## Theme system (v0.1 系 → v0.14 で安定済)

8 theme (4 family × light/dark = `mint` / `sora` / `contrast` / `oldschool` × `light` / `dark`) は v0.14 時点で `creoui/tokens.css` 内に同梱済、 `[data-theme="<id>"]` で切替。 v0.17 では touch なし。 詳細は [`docs/design/theme-system.md`](../design/theme-system.md)。

## creoui-editor-host @0.4.0 → @0.5.0

editor-host も同期して bump (v0.5.0 published)。 主要変更:

### Public type re-exports (additive)

```ts
// v0.5.0 から annotate 可能
import type {
  EditorHostConfig,
  EditorShortcut,
  EditorHost,
  EditorHostMcpApi,
  EditorField,
  EditorFieldType,
  EditorFieldConstraints,
} from 'creoui-editor-host'
```

### `exposeConsole` の default behavior change (consumer-actionable)

```diff
- production build でも window.creoEditor が expose されていた (v0.4.x)
+ default が import.meta.env.DEV に追従、 production では false (v0.5.0+)
```

production deploy で `window.creoEditor` を意図的に expose したい場合は明示的に渡す:

```tsx
<EditorHostProvider config={{ exposeConsole: true }}>
```

CLAUDE.md EH-6 規定 (`Console REPL を production で無条件に expose しない`) への準拠です。

## Recommended migration steps

1. **install bump**:
   ```bash
   bun add creoui@^0.18 creoui-editor-host@^0.5
   ```

2. **(optional) soft-landing**: `creoui/token-shim.css` を import して旧 `var(--margin-md)` 等の CSS variable 名で動かしながら段階移行

3. **bulk CSS sed sweep** (上記 18 行の sd command — token rename)

4. **bulk HTML attribute sed sweep** (上記 8 行の sd command — `data-size` / `data-padding` / `data-elevation`)

5. **JS const import rename** (camelCase identifier)

6. **Swift / Rust source も同 rename** (使っている場合)

7. **production で editor-host を expose したい consumer** は `<EditorHostProvider config={{ exposeConsole: true }}>` を明示

8. **typecheck + visual smoke test**:
   ```bash
   bun run typecheck
   bun run dev   # 視覚確認
   ```

9. **deploy** (a11y reduced-motion は consumer 側追加作業なしで効きます)

## Stability commitment (v0.18 以降)

v0.18 は **5 tier convention 完全統一 (token + attribute) の最終 release**。 「sm/md/lg」 由来の breaking rename は **これ以上発生しない予定** (`docs/foundations/principles` 原則 01 として articulate)。 以降の breaking change は新 capability 追加時のみで、 同 token name / attribute value の rename は発生しない設計。
